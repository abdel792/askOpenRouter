# Запитай OpenRouter

* Автор(и) : Abdel.

Це доповнення для NVDA дозволяє взаємодіяти з моделями штучного інтелекту, наданими платформою OpenRouter, безпосередньо з вашого екранного читача.

Доповнення підтримує обидва варіанти:
* Автоматичний випадковий вибір безкоштовних моделей
* Ручний вибір будь-якої доступної моделі (включаючи платні)

## Основні можливості

* Швидкий доступ: Відкривайте інтерфейс чату в будь-який час за допомогою глобального комбінації клавіш.
* Керування розмовами: Починайте нову розмову або продовжуйте попередній обмін повідомленнями.
* Розумна ротація безкоштовних моделей: Автоматично вибирає випадкову безкоштовну модель для оптимізації щоденних квот використання.
* Ручний вибір моделі: Вибирайте конкретну модель (включаючи платні моделі) на панелі налаштувань.
* Доступні результати: Переглядайте відповіді у чіткому вікні з легкою навігацією та можливістю відображення повної історії.

## Налаштування: Отримання та встановлення вашого API ключа

Щоб використовувати це доповнення, ви повинні мати API ключ від OpenRouter.

Навіть при використанні безкоштовних моделей ключ необхідний для ідентифікації ваших запитів.

### 1. Як отримати API ключ

1. Перейдіть на [OpenRouter.ai](https://openrouter.ai/).
2. Створіть обліковий запис, натиснувши "Sign up" (ви можете увійти за допомогою облікового запису GitHub, Google або MetaMask, або вашої електронної адреси).
3. Після входу перейдіть до розділу "Keys" у вашій панелі керування або перейдіть безпосередньо за посиланням: https://openrouter.ai/keys
4. Натисніть кнопку "Create Key".
5. Дайте вашому ключу назву (наприклад: "Мій API ключ OpenRouter") і натисніть "Create".
6. Важливо: Ваш ключ буде показано лише один раз. Скопіюйте його негайно та збережіть у безпечному місці.

### 2. Налаштування ключа в NVDA

1. Відкрийте меню NVDA (NVDA + N).
2. Перейдіть до Параметри, потім Налаштування.
3. У списку категорій виберіть "Ask OpenRouter".
4. Вставте свій API ключ у поле "OpenRouter API Key".
5. Натисніть OK для збереження.

#### Показувати API ключ

На панелі налаштувань NVDA, одразу після поля "OpenRouter API Key", є прапорець з написом:

"Show API key"

Якщо його позначено, символи API ключа стають видимими.
За замовчуванням вони приховані з міркувань безпеки.

## Налаштування вибору моделі

У категорії налаштувань Ask OpenRouter ви знайдете нову опцію:

### "Use all models, including paid ones"

Ця опція контролює спосіб вибору моделей.

### Коли опцію НЕ ПОЗНАЧЕНО (поведінка за замовчуванням)

* Доповнення автоматично вибирає випадкову безкоштовну модель для кожної нової розмови.
* Воно чергує доступні безкоштовні моделі.
* Це допомагає розподілити використання та уникнути лімітів на частоту запитів.

### Коли опцію ПОЗНАЧЕНО

Коли цю опцію увімкнено, список доступних моделей автоматично з'являється після прапорця.

* Список відсортований у порядку зростання ціни за токени промпту (вартість за вхідний токен), від найнижчої до найвищої.
* Відображаються лише застарілі моделі з діючими провайдерами.

### Що ви можете робити, коли цю опцію увімкнено?

* Вибирати будь-яку доступну модель.
* Використовувати платні моделі (якщо у вас достатньо кредитів OpenRouter).
* Вибирати модель, яка найкраще відповідає вашим потребам.
* Продовжувати використовувати ту саму вибрану модель для ваших розмов (без автоматичної ротації).

### Що таке токен промпту?

Токен промпту представляє собою невелику одиницю тексту, надіслану моделі (ваше запитання або вхідні дані).

Рахунки за моделі зазвичай виставляються окремо за:
* Вхідні токени (промпт)
* Вихідні токени (відповідь)

## Як використовувати

### Відкриття діалогу чату

Натисніть:

Ctrl + Alt + A

Ви можете змінити цей жест у:
Меню NVDA → Параметри → Жести вводу → Ask OpenRouter

### Головний інтерфейс

Діалог містить три кнопки:

1. New Chat – Починає абсолютно нову розмову.
2. Continue Chat – Відновлює попередню розмову (зберігає історію).
3. Close – Закриває діалог (Escape також працює).

### Введення вашого промпту

Після вибору "New Chat" або "Continue Chat":

* З'являється багаторядкове текстове поле.
* Натискання Enter вставляє новий рядок.
* Щоб надіслати повідомлення:
  - Натисніть Tab, щоб дійти до кнопки OK.
  - Натисніть Enter.

### Читання відповіді

Після обробки з'являється вікно результатів, що містить:

* "You said:" після чого йде ваше повідомлення.
* "The model replied:" після чого йде відповідь.
* Кнопку "Copy" для копіювання відповіді.

Якщо увімкнено відображення повної історії, кожен обмін повідомленнями чітко розділений заголовками, що полегшує навігацію за допомогою клавіш швидкої навігації NVDA.

## Параметри відображення

Якщо ви віддаєте перевагу відображенню лише останньої відповіді замість повної історії розмови:

1. Відкрийте меню NVDA (NVDA + N).
2. Перейдіть до Параметри → Налаштування.
3. Виберіть Ask OpenRouter.
4. Зніміть прапорець:
   "Display the full chat history for continuous discussions"
5. Натисніть OK.

## Непризначені скрипти

Наступні скрипти не мають призначених жестів.
Ви можете визначити їх у:

Параметри → Жести вводу → Ask OpenRouter

Доступні скрипти:

* Відкрити панель налаштувань доповнення
* Почати новий чат безпосередньо
* Продовжити існуючий чат безпосередньо

## Безкоштовні моделі, платні моделі та квоти

### Використання безкоштовних моделей

Коли "Use all models, including paid ones" не позначено:

* Використовуються лише моделі, позначені як безкоштовні на OpenRouter.
* Безкоштовні моделі мають:
  - Обмежені щоденні квоти
  - Спільні ліміти на частоту запитів
  - Можливу тимчасову недоступність

Доповнення автоматично чергує безкоштовні моделі для покращення доступності.

### Використання платних моделей

Коли "Use all models, including paid ones" позначено:

* Доповнення використовує саме ту модель, яку ви вибрали.
* Це може включати платні моделі.
* Ви повинні мати достатньо кредитів OpenRouter.
* Можуть застосовуватися ліміти частоти запитів провайдера.

Помилки, такі як:
* 402 (недостатньо кредитів)
* 429 (ліміт запитів вичерпано)
* 404 (модель не дозволена налаштуваннями конфіденційності)

відображаються безпосередньо, щоб інформувати вас про проблему.

## Нагадування про налаштування конфіденційності

Якщо ви використовуєте безкоштовні моделі та отримуєте помилку з згадкою:

> "No endpoints found matching your data policy"

Можливо, вам потрібно налаштувати параметри конфіденційності OpenRouter:

https://openrouter.ai/settings/privacy

Переконайтеся, що публічні/безкоштовні кінцеві точки моделей дозволені.
